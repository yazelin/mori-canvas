＃這是什麼

一個自架、零授權成本(全 MIT)的 AI 會議白板。核心是:把「你講的話」交給 AI,它判斷這句是會議內容還是給它的指令,然後自動在白板上長出便利貼、連線、分區。所有人開瀏覽器連同一個房號就能即時一起看、一起編。

所有重活(語音轉文字、AI 整理、白板狀態)都在「主機」那一台,其他人只是用瀏覽器連進來,零安裝。便利貼會依性質自動上色:

主題 待辦 決議 風險

＃30 秒上手

  1. 開 試玩連結(或你自架的網址),輸入名字進入會議。
  2. 按左下角的 「開始會議記錄」(紅點按鈕),允許麥克風,然後就正常講話。
  3. 講一段、停頓一下,AI 就把那段重點整理成便利貼上板。整場 hands-free。不想講話?也可把現成逐字稿貼進面板按「丟給 agent」。

要拉人一起:工具列「分享 / QR」→ 同事掃 QR 或輸入房號就進來,大家即時同步。

＃影片介紹

先看實際流程:貼上逐字稿讓 AI 長出白板,再把成果匯出成可閱讀、可分享的檔案。

一開會記錄(語音)

這是最主要的用法。左下「開會記錄」面板:

  • 開始會議記錄:連續錄音,偵測到你停頓會自動斷句、送轉錄 + AI 整理上板;再按一次停止。整場邊講邊長。
  • 單次錄一段:只錄一段(按開始、講話、再按停止),適合補一句。
  • 貼現成的逐字稿:把別處的會議逐字稿貼進來,按「丟給 agent」一次整理。
  • 逐字記錄:你講過的每一段都會一行行累積在面板裡(全程 word-for-word),也會餵給 AI 當上下文,讓它整理得更準。跟著畫板一起存檔。

辨識出的文字會在畫面下方浮現幾秒(給說話的人提示);多人同時錄沒問題,卡片會累積、不互蓋。語音輸出一律自動轉成繁體中文(台灣用語)。

二語音指令大全

AI 會分辨你這句是「會議內容」還是「操作白板的指令」。下面這些直接用講的就會執行,不用找按鈕:

你說它做什麼
幫我排一下 / 排版把每張圖依板型重新排整齊
只看亞澤的只顯示某個人負責的卡
只看 #行銷只顯示某個標籤的卡
顯示全部取消篩選
把這張交給小明指派負責人(會抽出 amber 名牌)
把定價那張改成決議改卡片類型(顏色)
把官網那張改寫成 RWD 重構改寫卡片文字
把 3 號連到 5 號把兩張既有卡畫上關係箭頭
開三個區:臨時動議、會議進程、待討論一次開好幾個命名分區
庫存那張討論完了,移到已討論把卡片搬到另一個區

拿不準時它會當成「會議內容」建卡,不會亂。

三AI 認得你指的是哪一張卡

你不用先用滑鼠點選某張卡。直接講「定價那張」「剛剛那張」「關於庫存的」,AI 會從現況用內容比對找出是哪一張,然後改那一張,而不是重複建一張新的。

萬一它認錯,每張卡右上角有個小編號當精準備案 —— 你可以說「把 3 號移到已討論」「把 5 號改成風險」。

四命名區 + 搬卡(會議主持)

把白板分成幾個「區」(圖框),讓進度看得見:

  1. 主持人說「開三個區:臨時動議、會議進程、待討論」→ 一次開好三個命名區。
  2. 大家講內容,AI 自動把項目放進對的區(說「待討論的項目:官網改版、員工旅遊」就進「待討論」)。
  3. 討論到了就說「官網改版討論完了,移到已討論」→ 卡片跳到「已討論」區,進度一目了然。

你也可以手動把卡片拖進某個圖框,它就歸到那張圖。

五多人協作 / 分享

進場報名

每個人一進來先輸入名字,卡片與游標就標得出是誰提的。

分享 / QR

工具列「分享 / QR」→ QR、房號(短代碼)、邀請連結、進行中房間清單。掃 QR 或輸入房號就加入。

即時同步

白板任何變動(AI 上板、別人拖卡、改字)立刻同步給所有人;看得到彼此的游標,也看得到 Mori 一張張畫。

零安裝

其他人只要瀏覽器,不用裝任何東西。他們講的話用主機資源處理。

七板型(10 種)

工具列徽章可切板型。板型決定 AI 怎麼解讀卡片與連線、怎麼配色、怎麼自動排版 —— 同一個引擎,換板型就畫不同的圖:

會議白板(分欄)、組織架構圖、流程圖、系統架構圖、心智圖、看板 Kanban、SWOT、時間軸、魚骨圖、甘特圖。一個會議的畫布上可以有多張圖,AI 切到新主題時會自動開對應的圖。

九設定(語音 / AI)

語音轉文字(STT)三條路

模式說明
Mori 處理委派 mori-ear(本機 whisper / Groq 由它決定)。需裝過 mori-ear。
自訂 · 雲端Groq Whisper(whisper-large-v3-turbo),填自己的 Groq key。客戶零安裝首選。
自訂 · 本機打一台本機 whisper-server,資料不出網。預設模型 small,有顯卡可換 large-v3-turbo。

AI(把逐字稿整理成卡片)

預設 Groq openai/gpt-oss-120b(雲端),連不到時退本機 Ollama qwen3。

用你自己的 AI(BYO)

設定頁可填任何 OpenAI 相容的 base / key / model(OpenAI、Gemini、Azure、Groq、本機 Ollama 都行),就用你自己的額度、不耗主機。key 只存你瀏覽器。

Groq API Key

沒裝 mori-ear、環境也沒設 key 時,設定頁可直接貼上 Groq key(console.groq.com/keys 免費申請),就解鎖雲端語音與 AI。

外觀

右上(手機在 ⋯ 選單)可切深 / 淺色主題,首次會跟著你系統的設定。

十加到主畫面(PWA)

它是 PWA,可以裝成像原生 App、全螢幕、沒有瀏覽器網址列:

  • Android / Chrome / Edge:⋯ 選單會出現「安裝 App」,或網址列的安裝圖示。
  • iPhone / Safari:點下方「分享」→「加入主畫面」。

十一三種用法 / 自架

① 試玩(免裝)

直接開 試玩連結。AI 走站長的 key(有限流);想用自己的額度就在設定填 BYO。

② 自己跑 server

給團隊、想穩定用、資料留自己這。別人連你的區網或網址。

③ 桌面 App

GitHub Releases 下載 Windows / Linux 安裝檔,雙擊就開原生視窗。

自架 server(clone 回本地玩)

git clone https://github.com/yazelin/mori-canvas
cd mori-canvas
npm install && npm run build      # 編前端 + Rust binary(前端內嵌進去)
# 準備一把 Groq key:環境變數 GROQ_API_KEY,或同資料夾放 .env(GROQ_API_KEY=...)
./server-rs/target/release/mori-canvas-server   # 開 http://localhost:1334/

需要 Rust(cargo) 跑、Node 只用來 build 前端。要讓同網段的人一起開會(手機要錄音需 HTTPS):npm run setup 產自簽憑證後 npm run start:lan,大家連 https://你的區網IP:5174/。完整說明見 repo README。

也能一鍵部署到 Render(連 GitHub repo,讀 render.yaml,後台填 GROQ_API_KEY),或裝進 AgentOS 當 body-part。

＃隱私與授權

  • 自架時資料是你的:房間存在主機的 .data/,不對外傳。逐字稿、卡片都在你機器上。
  • 試玩版(Render 免費方案):閒置會休眠、重啟會清空 —— 想留請「下載畫板檔」。AI 走站長的 key、有 per-IP 限流。
  • 授權:全 MIT(後端 Rust:yrs / warp / tokio;前端:yjs / konva / react / vite)—— 可閉源、可賣、可改。

？常見問題

試玩版第一次很慢?

Render 免費方案閒置 15 分會休眠,第一個進來的人要等 ~30–60 秒冷啟動,之後就正常。要穩定就自架或用付費方案。

會不會用到我的 Groq 額度?

試玩版用站長的。你自架就用你自己的 key;或在設定填「用你自己的 AI(BYO)」,key 只存你瀏覽器。

一定要 mori-ear 嗎?

不用。設定切「自訂 · 雲端」用 Groq Whisper(填 key)就能語音;完全不想語音,打字 / 貼逐字稿也行。

資料會不會外洩?

自架的話所有處理都在你主機。要完全不出網:設定走「自訂 · 本機 whisper」+ 本機 Ollama,或開 LLM_LOCAL_ONLY=1。

跑出簡體字?

不會 —— 輸出都會用程式硬轉成繁體(台灣用語)。若真的看到,GitHub 開 issue 跟我說。

桌面版跟 server 版差在哪?

桌面 App 是單機(loopback);要多人一起開會請用「自己跑 server」(對區網開放,別人連得進來)。
